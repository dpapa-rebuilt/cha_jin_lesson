# Donnycraft PRD (Product Requirements Document)

## 1. 서비스 개요

**서비스명:** Donnycraft  
**URL:** jinzthink.com/donnycraft (예정), cha-jin-lesson.vercel.app  
**성격:** 가족 전용 미션 보상 서비스 (비공개)

아이의 학습 및 성장을 돕기 위해 아빠가 미션을 설계하고, 아이가 미션을 수행하면 포인트가 쌓이는 서비스. 아이는 모인 포인트를 원하는 것과 교환할 수 있고, 아빠가 실제로 사줄 때 포인트를 차감한다.

---

## 2. 사용자

| 역할 | 접근 방식 | 주요 행동 |
|------|-----------|-----------|
| **아이** | 비밀번호 없이 바로 접속 | 미션 확인, 완료 신청, 포인트 잔액 확인 |
| **아빠** | 비밀번호 입력 후 관리자 모드 진입 | 미션 생성/수정, 완료 승인/거절, 포인트 차감 |

---

## 3. 핵심 기능

### 3.1 미션 시스템

#### 미션 종류
| 유형 | 설명 | 예시 |
|------|------|------|
| **1회성** | 특정 날짜까지 한 번만 수행 | 수학 문제집 1단원 끝내기 |
| **매일** | 매일 반복 | 지문 읽고 문제 풀기 |
| **매주** | 특정 요일마다 반복 | 매주 수요일 일기 쓰기 |
| **매월** | 특정 주·요일마다 반복 | 매월 마지막 주 일요일 아빠와 아웃도어 액티비티 |

#### 미션 속성
- 제목, 설명(선택), 유형, 보상 포인트, 반복 주기 또는 마감일
- 활성화/비활성화 토글 가능

### 3.2 완료 신청 흐름 (아이)

1. 홈 화면에서 오늘 해야 할 미션 확인
2. "완료했어요!" 버튼 클릭
3. 어떻게 했는지 메모 입력 (선택 사항)
4. "보내기" 클릭 → "승인 기다리는 중..." 상태로 변경

#### 완료 상태
| 상태 | 표시 |
|------|------|
| 대기 중 | "승인 기다리는 중..." (노란색) |
| 승인됨 | "완료! 포인트 적립됨" (초록색) |
| 거절됨 | "다시 도전하기" 버튼 표시 |

### 3.3 승인 흐름 (아빠)

1. 승인 대기함에서 아이의 완료 신청 확인
2. 아이가 작성한 메모(💬 말풍선) 확인
3. **승인:** 포인트 즉시 적립
4. **거절:** 거절 이유 입력(선택) 후 거절 확정

### 3.4 포인트 시스템

- 미션 승인 시 포인트 자동 적립
- 포인트는 서비스 내 가상 화폐 (실제 현금 아님)
- 아이가 원하는 것을 사줄 때 아빠가 내용 기록 + 포인트 차감
- 잔액은 아이 홈 화면 상단에 항상 표시

## 4. 화면 구성

### 아이 화면 (비밀번호 없음)
| 화면 | 경로 | 내용 |
|------|------|------|
| 홈 | `/` | 포인트 잔액, 오늘의 미션, 다른 미션 목록 |

### 아빠 화면 (비밀번호 필요)
| 화면 | 경로 | 내용 |
|------|------|------|
| 로그인 | `/admin` | 비밀번호 입력 |
| 대시보드 | `/admin/dashboard` | 잔액, 승인 대기 수, 전체 미션 목록, 최근 내역 |
| 미션 만들기 | `/admin/missions/new` | 미션 신규 생성 폼 |
| 미션 수정 | `/admin/missions/[id]/edit` | 기존 미션 편집 |
| 승인 대기함 | `/admin/approve` | 완료 신청 목록, 승인/거절 |
| 포인트 사용 | `/admin/spend` | 포인트 차감 및 사용 내역 |

## 5. 데이터 구조

### missions
| 필드 | 타입 | 설명 |
|------|------|------|
| id | integer | 고유 식별자 |
| title | varchar | 미션 이름 |
| description | text | 미션 설명 (선택) |
| type | varchar | once / daily / weekly / monthly |
| recurrence_rule | jsonb | 반복 규칙 (요일, 주차 등) |
| points | integer | 보상 포인트 |
| is_active | boolean | 활성화 여부 |
| due_date | date | 마감일 (1회성 미션) |

### completions
| 필드 | 타입 | 설명 |
|------|------|------|
| id | integer | 고유 식별자 |
| mission_id | integer | 연결된 미션 |
| status | varchar | pending / approved / rejected |
| note | text | 아이가 쓴 완료 메모 |
| reject_reason | text | 아빠의 거절 이유 |
| completed_at | timestamptz | 완료 신청 시각 |
| approved_at | timestamptz | 승인 시각 |

### transactions
| 필드 | 타입 | 설명 |
|------|------|------|
| id | integer | 고유 식별자 |
| type | varchar | earn (적립) / spend (사용) |
| points | integer | 포인트 수량 |
| description | text | 내용 설명 |
| completion_id | integer | 연결된 완료 기록 (적립 시) |
| created_at | timestamptz | 발생 시각 |
