# Donnycraft — 제품 요구사항 문서 (PRD)

> 아이의 학습과 성장을 돕는 가족 전용 미션 보상 서비스

---

## 1. 제품 개요

**Donnycraft** 는 아빠가 아이에게 미션을 설계하고, 아이가 미션을 완료하면 포인트가 쌓이는 가족 전용 웹 서비스입니다. 아이는 모인 포인트로 원하는 것을 교환할 수 있고, 아빠가 실제로 사줄 때 포인트를 차감해 기록합니다. 결제·현금 없이 가상 포인트로만 운영합니다.

**URL:** `jinzthink.com/donnycraft` (예정) / `cha-jin-lesson.vercel.app`  
**성격:** 가족 전용 비공개 서비스 (아빠 1명, 아이 1명)

---

## 2. 사용자와 핵심 사용 시나리오

### 사용자 (역할 2종)

| 역할 | 접근 방식 | 주요 행동 |
|------|-----------|-----------|
| **아이** | 비밀번호 없이 바로 접속 | 미션 확인, 완료 신청, 포인트 잔액 확인 |
| **아빠** | 비밀번호 입력 후 관리자 모드 | 미션 생성·수정, 완료 승인·거절, 포인트 차감 |

### 핵심 사용 시나리오

**시나리오 1 — 아이가 오늘 미션을 확인하고 완료를 신청한다**
아이가 홈 화면에 접속하면 오늘 해야 할 미션 목록과 현재 포인트 잔액이 보인다. "글 읽고 답하기" 미션의 "완료했어요!" 버튼을 누르고, "오늘 3페이지 읽고 문제도 다 풀었어요!"라고 메모를 쓴 뒤 보낸다. 버튼이 "승인 기다리는 중..."으로 바뀐다.

**시나리오 2 — 아빠가 완료를 확인하고 포인트를 적립한다**
아빠가 관리자 화면에 접속해 승인 대기함을 연다. 아이가 쓴 메모를 읽고 내용이 맞다고 판단해 "승인하기"를 누른다. 포인트가 즉시 아이 잔액에 더해진다. 내용이 부족하다면 "거절하기"를 누르고 "다음엔 문제 번호도 적어줘"라고 이유를 남긴다.

**시나리오 3 — 아이가 포인트로 원하는 것을 교환한다**
아이가 레고 세트를 갖고 싶다고 했다. 아빠가 포인트 사용 화면에서 "레고 테크닉 42151"이라고 입력하고 500포인트를 차감한다. 아이 잔액에서 500포인트가 빠지고 사용 내역에 기록이 남는다.

**시나리오 4 — 아빠가 새 반복 미션을 만든다**
아빠가 미션 만들기 화면에서 "매월 마지막 주 일요일 아빠와 아웃도어 액티비티"를 만든다. 유형은 "매월", 마지막 주 일요일을 설정하고 보상 포인트를 100으로 지정한다. 다음 해당 날짜가 되면 아이 홈 화면에 자동으로 나타난다.

## 4. 각 기능의 수용 기준 (이렇게 작동하면 완성)

### F1. 아빠 비밀번호 인증
- [x] 비밀번호를 맞게 입력하면 관리자 화면으로 이동한다.
- [x] 틀리면 "비밀번호가 틀렸어요" 메시지가 표시되고 다시 시도할 수 있다.
- [x] 로그인 상태는 30일간 유지된다 (쿠키).
- [x] 로그아웃하면 관리자 화면에 접근할 수 없고 로그인 화면으로 이동한다.

### F2. 미션 생성
- [x] 제목, 설명(선택), 유형, 보상 포인트를 입력해 미션을 만들 수 있다.
- [x] 유형은 1회성·매일·매주·매월 중 하나를 선택한다.
- [x] 매주는 요일을, 매월은 주차+요일을 추가로 지정한다.
- [x] 1회성 미션은 마감일을 설정할 수 있다.
- [x] 저장하면 대시보드 미션 목록에 즉시 반영된다.

### F3. 미션 수정 및 관리
- [x] 대시보드에서 모든 미션의 제목·유형·포인트를 한눈에 볼 수 있다.
- [x] 수정 버튼을 누르면 기존 값이 채워진 편집 폼이 열린다.
- [x] 비활성화하면 아이 화면에서 사라지고, 다시 활성화하면 나타난다.

### F4. 오늘의 미션 자동 노출
- [x] 매일 미션은 매일 표시된다.
- [x] 매주 미션은 설정한 요일에만 표시된다.
- [x] 매월 미션은 설정한 주차·요일에만 표시된다 (예: 마지막 주 일요일).
- [x] 1회성 미션은 마감일 이전, 미완료 상태일 때 표시된다.
- [x] 오늘 해당하지 않는 미션은 "다른 미션들" 섹션에 따로 보여 구분된다.

### F5. 완료 신청
- [x] 아이가 "완료했어요!" 버튼을 누르면 메모 입력창이 나타난다.
- [x] 메모는 선택 사항이며, 쓰지 않아도 보낼 수 있다.
- [x] 신청 후 버튼이 "승인 기다리는 중..."으로 바뀐다.
- [x] 같은 기간 내 이미 신청했으면 버튼이 비활성화 상태로 유지된다.
- [x] 거절된 미션은 "다시 도전하기" 버튼이 나타나 재신청할 수 있다.

### F6. 완료 승인·거절
- [x] 아빠 승인 대기함에 아이의 미션 이름, 신청 시각, 메모(💬)가 표시된다.
- [x] 승인하면 포인트가 즉시 적립된다.
- [x] 거절하기를 누르면 이유 입력창이 펼쳐진다 (선택 사항).
- [x] 거절 확정 후 해당 완료 건은 대기함에서 사라진다.

### F7. 포인트 자동 적립
- [x] 승인 즉시 해당 미션의 보상 포인트가 잔액에 더해진다.
- [x] 아이 홈 화면 잔액이 실시간으로 반영된다.
- [x] 적립 내역이 대시보드 최근 내역에 기록된다.

### F8. 포인트 차감
- [x] 아빠가 무엇을 사줬는지 설명과 차감할 포인트를 입력한다.
- [x] 차감 후 잔액에서 즉시 빠진다.
- [x] 사용 내역에 날짜·내용·포인트가 기록된다.

### F9. 포인트 잔액 확인
- [x] 아이 홈 화면 상단에 현재 잔액이 항상 표시된다.
- [x] 아빠 대시보드에도 현재 잔액이 표시된다.
- [x] 잔액 = 총 적립 포인트 − 총 사용 포인트.

### F10. 역할별 대시보드
- [x] 아이 홈: 현재 포인트 잔액 + 오늘의 미션 + 다른 미션들.
- [x] 아빠 대시보드: 잔액, 승인 대기 건수, 전체 미션 목록, 최근 포인트 내역, 바로가기.

## 5. 성공 지표

이 서비스가 잘 작동하고 있다는 것을 다음으로 판단한다.

- **지속 사용**: 아이가 매일 화면을 열어 미션을 확인한다.
- **완료율**: 오늘의 미션 중 실제로 완료 신청하는 비율이 70% 이상.
- **승인 속도**: 아빠가 완료 신청을 24시간 이내에 처리한다.
- **포인트 순환**: 적립된 포인트가 실제로 교환·차감되어 잔액이 무한정 쌓이지 않는다.
- **미션 다양성**: 1회성·반복 미션이 혼합되어 아이가 다양한 목표를 갖는다.

## 6. 범위 밖 (Out of Scope)

다음은 이번 버전에서 **만들지 않는다**.

- 실제 현금 결제·포인트 환전
- 아빠 이외의 보호자(엄마 등) 복수 관리자
- 여러 자녀 지원 (현재는 아이 1명 기준)
- 이메일·문자·푸시 알림
- 아이의 포인트 사용 신청 (아이가 직접 차감 요청하는 기능)
- 미션 완료 사진 첨부
- 통계·리포트 차트
- 모바일 전용 앱 (반응형 웹으로만 대응)

## 7. 가정 (Assumptions)

설계 중 확정되지 않아 다음과 같이 가정했습니다. 사실과 다르면 수정이 필요합니다.

1. 사용자는 **아빠 1명, 아이 1명**이며 별도 회원가입 없이 운영한다.
2. 아이는 **공유 기기**(태블릿·PC)로 접속하며, 아빠 비밀번호를 모른다.
3. 포인트는 **가상 화폐**이며 실제 현금과 연동되지 않는다.
4. 미션 완료 여부는 **아빠가 직접 판단**하며, 자동 검증 기능은 없다.
5. 데이터는 클라우드(Neon DB)에 저장되어 기기에 상관없이 접근 가능하다.
6. 반복 미션의 "완료 기간"은 유형별로 다르다: 매일→당일, 매주→해당 주, 매월→해당 월.

## 8. 데이터 구조

### missions
| 필드 | 타입 | 설명 |
|------|------|------|
| id | integer | 고유 식별자 |
| title | varchar | 미션 이름 |
| description | text | 미션 설명 (선택) |
| type | varchar | once / daily / weekly / monthly |
| recurrence_rule | jsonb | 반복 규칙 `{dayOfWeek}` 또는 `{week, dayOfWeek}` |
| points | integer | 보상 포인트 |
| is_active | boolean | 활성화 여부 |
| due_date | date | 마감일 (1회성 미션) |
| created_at | timestamptz | 생성 시각 |

### completions
| 필드 | 타입 | 설명 |
|------|------|------|
| id | integer | 고유 식별자 |
| mission_id | integer | 연결된 미션 |
| status | varchar | pending / approved / rejected |
| note | text | 아이가 쓴 완료 메모 |
| reject_reason | text | 아빠의 거절 이유 |
| completed_at | timestamptz | 완료 신청 시각 |
| approved_at | timestamptz | 승인 시각 |

### transactions
| 필드 | 타입 | 설명 |
|------|------|------|
| id | integer | 고유 식별자 |
| type | varchar | earn (적립) / spend (사용) |
| points | integer | 포인트 수량 |
| description | text | 내용 설명 |
| completion_id | integer | 연결된 완료 기록 (적립 시) |
| created_at | timestamptz | 발생 시각 |
