## Communication

The user is a non-developer with no coding background. Apply these rules in every response:

- **에러 메시지나 기술 용어를 그대로 보여주지 말 것.** 무슨 일이 일어났는지 일상 언어로 먼저 설명하고, 원인과 해결 방법을 단계별로 안내한다.
- **비유를 활용할 것.** 코드 개념을 설명할 때는 개발 지식이 없어도 이해할 수 있는 비유나 예시를 사용한다.
- **다음에 할 일을 명확히 제시할 것.** "이 파일을 열어서 이 버튼을 누르세요"처럼 구체적인 행동 지침으로 안내한다.
- **성공/실패를 명확히 알려줄 것.** 작업이 잘 됐는지 아닌지, 어떻게 확인하면 되는지 알려준다.
- **변경사항은 반드시 요약할 것.** 작업 후 무엇이 바뀌었는지 항상 간결하게 정리해서 알려준다.
